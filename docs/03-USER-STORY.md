# 3. 使用情境 (User Story)

> 維護時可引用：**驗收條件、測試情境、產品／活動說明**。

---

## 情境摘要

在公益活動現場，志工透過平板輸入孩童的夢想與基本資訊，系統經由 AI 生成「未來職人」情境圖與鼓勵小語，並可列印贈送給孩童。

---

## 詳細流程

1. **場景**  
   綠色奇蹟的電腦捐贈活動現場。

2. **互動**  
   志工拿著平板問小朋友：「你長大想做什麼？」

3. **輸入**  
   小朋友說：「我想當醫生，救很多人。」  
   志工在系統輸入「醫生」、「男生」等資訊。

4. **AI 運作**
   * **Gemini**：將描述擴寫為繪圖用 Prompt，例如：「一位穿著白袍的亞洲小男孩醫生，脖子上掛著聽診器，背景是明亮溫暖的醫院，風格為皮克斯動畫風……」
   * **Replicate**：依描述生成對應情境圖像。

5. **結果**  
   螢幕顯示高品質圖片，並附上鼓勵小語，例如：「未來的名醫 [名字]，你的雙手將帶來奇蹟！」

6. **結局**  
   現場可列印贈送給小朋友，作為書桌上的激勵。

## 情境變體：自拍＋職業 → 25 歲擬真職業照（Web / LINE）

若改為**小朋友上傳自拍照**與**希望職業**，系統以臉部特徵生成**25 歲擬真職業照**。流程為：**① 年齡變化（SAM）→ ② Prompt 生成（Gemini）→ ③ 職業照生成（InstantID）**，詳見 [09-PRODUCT-SERVICE-STEPS.md](09-PRODUCT-SERVICE-STEPS.md)。

| 入口 | 說明 |
|------|------|
| **Web 版** | 開啟 `/career_photo`，上傳照片或貼網址、選擇職業，約 90–150 秒後顯示擬真職業照 |
| **LINE 官方帳號** | 使用者在 LINE 傳自拍＋職業 → Make 呼叫 API → 回傳圖片，見 [07-LINE-AND-FACE-SCENARIO.md](07-LINE-AND-FACE-SCENARIO.md) |
